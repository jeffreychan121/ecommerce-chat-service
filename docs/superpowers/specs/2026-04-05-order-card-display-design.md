# 订单卡片与消息展示优化设计

## 需求概述

1. **订单列表实时刷新** - 卡片下单后，订单列表立即显示新订单
2. **消息卡片保持一致** - 聊天窗口和历史记录都正确渲染商品卡片，而非 JSON 原文

---

## 架构设计

### 问题 1: 订单列表实时刷新

**方案**: Socket 实时通知

```
用户点击下单 → 后端创建订单 → 通过 Socket 广播 'order-created' → 前端监听并刷新订单列表
```

**涉及文件**:
- `src/modules/order/order.service.ts` - 创建订单后 emit 事件
- `src/modules/chat/chat.gateway.ts` - 广播事件
- `frontend/src/components/OrderTest.tsx` - 监听 'order-created' 刷新列表
- `frontend/src/hooks/useChat.ts` - 监听事件并触发回调

---

### 问题 2: 消息卡片一致渲染

**数据流**:
1. 用户发送消息 → Dify 返回包含 JSON 的文本
2. 后端保存消息时，将 JSON 提取存储为 `card` 字段
3. 前端获取历史消息时，按 `card` 字段渲染卡片

**关键点**:
- 后端保存 AI 消息时，需要解析 JSON 并分离 content 和 card
- 前端 MessageBubble 已有 ProductCard 组件，只需确保 card 数据正确传递

## 数据结构

### 保存到数据库的消息结构

```typescript
interface ChatMessage {
  id: string;
  sessionId: string;
  senderType: 'USER' | 'AI' | 'AGENT';
  content: string;        // 纯文本（不含 JSON）
  card: {                 // 商品卡片数据（新增）
    type: 'product';
    products: Product[];
  } | null;
  messageType: string;
  createdAt: Date;
}
```

### API 返回格式

```json
{
  "id": "xxx",
  "sessionId": "xxx",
  "senderType": "AI",
  "content": "您好！推荐您这款产品...",
  "card": {
    "type": "product",
    "products": [
      {"sku_id": "ELEC001", "name": "智能降噪耳机", "price": 299, ...}
    ]
  },
  "createdAt": "2026-04-05T12:00:00Z"
}
```

## 实现步骤

### 1. 订单实时通知 (Socket)

1. **后端 OrderService 创建订单后 emit 事件**
2. **ChatGateway 广播 'order-created' 事件**
3. **前端 OrderTest 监听并刷新订单列表**

### 2. 消息卡片解析 (后端)

1. **修改 ChatService 保存 AI 消息时**:
   - 检测 content 中是否包含 `{"type":"products"`
   - 提取 JSON 部分作为 `card` 字段
   - 从 content 中移除 JSON，只保留纯文本

### 3. 前端渲染 (已有组件)

1. **MessageBubble 根据 card 字段渲染 ProductCard**
2. **MessageList 传递 card 数据**
3. **客服后台复用同一套组件**

---

## 预期效果

- 用户在聊天窗口点击"立即下单" → 订单列表立即显示新订单
- 刷新页面后，之前的商品推荐消息显示为卡片（不是 JSON 文本）
- 客服后台查看历史消息，同样显示为卡片形式
